# 播放列表微型指示器 — 设计文档

**日期**: 2026-05-27
**状态**: 已确认

---

## 背景

Mini Player 目前加载文件时会在后台构建同目录下的所有音频文件作为播放列表，但用户无从得知：
- 当前播放列表中有哪些歌曲
- 共有多少首
- 当前处于第几首

需要在不破坏 400×50 轻量化设计的前提下，提供可视化的播放列表信息。

## 方案选择

选择**微型指示器（方案 C）**：在迷你条上添加一个紧凑的 "3/12 ▾" 标签，点击弹出迷你列表，不占用额外空间。

---

## UI 设计

### 位置

指示器放在**循环模式按钮**和**进度条**之间的位置：

```
[⏮] [▶] [⏭] [🔁] [3/12 ▾] [========进度条========] [1:23/4:05] [🔊]
```

### 交互规则

| 操作 | 行为 |
|------|------|
| 左键点击指示器 | 弹出迷你播放列表 |
| 点击列表中的曲目 | 跳转到该曲目并播放 |
| 点击列表外空白处 | 关闭列表 |
| hover 指示器 | tooltip 显示当前完整歌名 |

**明确移除的功能**: 滚轮切歌（防止误触）。

### 迷你列表样式

- 深色半透明背景，与右键菜单风格一致
- 当前播放曲目高亮 + "▶" 前缀
- 显示曲目序号和 "歌手 - 歌名"
- 列表长度超过 8 首时截断，底部显示 "... 还有 N 首"
- 列表弹出在指示器下方

## 技术方案

### 改动范围

仅涉及两个文件：

**`mini_player/core/player.py`**（已存在的接口）
- `playlist` 属性 — 返回播放列表路径列表
- `current_index` 属性 — 返回当前索引
- 新增 `jump_to(index: int)` — 跳转到指定索引并播放

**`mini_player/ui/mini_bar.py`**
- 新增 `TrackIndicator` 类 — 可点击的播放计数标签（继承 QLabel），点击时弹出列表
- 新增 `TrackPopup` 类 — 弹出式曲目列表（继承 QFrame），参考现有右键菜单的样式
- 在 `_build_ui()` 中插入指示器，放在循环按钮和进度条之间

### 信号连接

```
TrackIndicator.clicked → 弹出/收起 TrackPopup
TrackPopup.track_selected(index) → MusicPlayer.jump_to(index)
MusicPlayer.track_changed → TrackIndicator.refresh(count, index)
```

### 滚动曲目时列表位置同步

切歌时（`jump_to` / `next` / `prev`），如果弹出列表可见，高亮条自动移动到新曲目位置。
